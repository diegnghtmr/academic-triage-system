# GGA Review Rules

Review only the staged files.

## Core Architecture
- Preserve dependency direction: `bootstrap -> infrastructure -> application -> domain`.
- `domain` and `application` must remain framework-free.
- Keep Spring, JPA, security, AI, and delivery concerns in `infrastructure` / `bootstrap`.
- Adapter, mapper, and config classes in infrastructure should be package-private unless public visibility is required by the framework.

## Build and Configuration
- Java 21, Spring Boot 3.5.x, Gradle Kotlin DSL multi-module build.
- Spring AI version is **1.1.4**.
- OpenAI-compatible AI providers (OpenAI/OpenRouter/Groq) are allowed through Spring AI's OpenAI stack.
- Default AI operation must remain optional; the system must still work without AI.
- Docker and Compose changes must use service-to-service networking (never `localhost` inside containers).

## Code Review Focus
- Catch real correctness, syntax, configuration, and runtime issues in staged files.
- Flag mismatches between tests and current implementation.
- Ignore local-only non-versioned artifacts and files outside the staged diff.
- Do not block on outdated historical blueprint assumptions that contradict the current staged code and validated runtime behavior.

## Output Contract
- First line must be exactly one of:
  - `STATUS: PASSED`
  - `STATUS: FAILED`
- If failed, list only concrete violations in staged files.
